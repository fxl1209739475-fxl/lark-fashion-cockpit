## 老板娘的问题

```
找出春夏款里售罄率不到 50% 的，按销量排序，告诉我哪些该清仓
```

> 这是真实电商运营场景：季节末看哪些款滞销需清仓。问题含 4 个维度：筛选（季节+售罄率+状态）、派生指标（售罄率 = 已实销/做货）、排序、决策推荐。

---

## Skill 跑了什么（4 步链）

### Step 1：自然语言 → 结构化 query

AI 把问题翻译成：
```json
{
  "table": "01_产品库",
  "filter": [
    {"field": "季节", "op": "is", "val": "春夏"},
    {"field": "状态", "op": "isNot", "val": "历史款"}
  ],
  "derived": "售罄率 = 已实销 / 做货数 × 100",
  "filter_after": "售罄率 < 50",
  "sort": "已实销 desc"
}
```

### Step 2：调 lark-cli 拉数据

```bash
lark-cli base +record-list \
  --base-token LWsdbVtIaa2MaDsANm3cNdYgn1j \
  --table-id tblrKzymnPPQ98ZX \
  --format json
# 拿到 8 款产品全字段
```

### Step 3：本地算售罄率 + 筛选 + 排序

PowerShell / Python 简单逻辑。**不用 base +data-query 的复杂 DSL，直接拉所有产品本地处理**——8 款数据量小，本地算更稳。

筛选结果：
| 款号 | 名称 | 季节 | 状态 | 售罄率 | 库存占用 |
|---|---|---|---|---|---|
| OUT-2024-OL | 品牌二-OL 极简通勤西服 | 春夏 | 在售 | **40%** | **¥82,620** |
| DRS-2024-LC | 品牌二-法式蕾丝吊带连衣裙 | 春夏 | 开发中 | 0% | ¥0 |

### Step 4：AI 智能识别 + 生成报告

AI 不只是给数据，**会智能识别业务情况**：

> "DRS-2024-LC 状态是开发中，实销 0 是正常（还没上架），不是真滞销。所以真正需要清仓的只有 1 款 OUT-2024-OL。"

这是**单纯查询做不到的判断**。

最后生成飞书 DocxXML 文档（含表格 + 红色洞察 + 6 项可勾选的行动建议）：

- ✅ 本周内挂"早春末场" 6.5 折活动（¥459 → ¥299）
- ✅ 主播直播间组合搭配主推
- ✅ 2 周后没清完进瑕疵清仓渠道
- ...
