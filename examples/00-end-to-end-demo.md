### 21:00 - 飞书 IM 链接踩坑（4 坑之一）

> 第一次发的 markdown IM 链接在飞书渲染成纯文本不可点。换 interactive 卡片 + button 元素后修复。

```bash
# ❌ 错误用法
lark-cli im +messages-send --markdown "[查看](https://...)"

# ✅ 正确用法
lark-cli im +messages-send --msg-type interactive --content '{
  "elements": [{
    "tag": "action",
    "actions": [{"tag": "button", "text": {...}, "url": "..."}]
  }]
}'
```

**踩坑沉淀进 [examples/04-im-card-best-practice.md](./04-im-card-best-practice.md)。**

## 🎯 演示证明的核心论点

### 1. Skill 不是"另一个多维表"
直接复制多维表给陌生人 ≠ Skill。Skill 是「让 agent 在多维表里干活」的能力包。

### 2. AI 决策能力 ≠ AI 数据查询
普通 BI 给查询结果，本 Skill 给「业务建议 + 优先级 + 可执行 todo」。AI 智能识别"开发中款不是滞销"是查询做不到的。

### 3. skill 间联动 > 单 skill 强大
1 个 skill 跑 1 件事是基础。**skill 编排能让"扫库存 → AI 决策 → 自动建任务 → 通知老板"链式跑通**。

### 4. 真朋友真任务真完成
- 30 个对手作品里 0 个能展示**跨租户外部联系人真收到任务真点完成**
- 这是参赛"业务增效"的最强证据

### 5. 工程性踩坑沉淀
4 个真实坑（task assignee 静默无效 / 跨租户 P2P 禁止 / P2P self 无推送 / markdown 链接失效）已固化进 SKILL.md，后续用户开箱即用。
