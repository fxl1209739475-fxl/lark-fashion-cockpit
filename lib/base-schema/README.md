# 飞书多维表 Schema 设计

> 14 张表 / 一个飞书多维表 App / 服务 19 大能力

## 设计原则

1. **单一 App，多表关联**：所有 14 张表装在一个飞书多维表 App 里，通过"双向关联"字段让表之间能跨表查询
2. **款号是中心**：产品库的"款号"是核心外键，几乎所有表都通过它关联到产品
3. **角色字段标准化**：所有任务相关表用统一的 9 角色枚举（设计师/打版师/生产主管/摄影师/模特/运营/内容编辑/主播/客服）
4. **状态字段统一**：所有流程类表用统一的状态枚举（待启动/进行中/已完成/已取消/已逾期）

## 字段类型说明

飞书多维表支持的字段类型：
- **文本**（多行/单行）
- **数字**（整数/小数/货币/百分比）
- **日期/时间**
- **单选/多选**（带颜色）
- **成员**（@人，存 open_id）
- **附件**（图片/文件）
- **关联**（双向关联其他表）
- **查找引用**（跨表读字段）
- **公式**（自动计算）
- **创建/修改时间/人**（自动）

## 关联关系图

```
products（款号）
   ├── 关联 → tasks（关联款）
   ├── 关联 → launch_waves（SKU 列表）
   ├── 关联 → stock_alerts（SKU）
   ├── 关联 → scripts（关联款 - 间接）
   ├── 关联 → live_schedule（主推款）
   ├── 关联 → production（在做款）
   ├── 关联 → returns（SKU）
   └── 查找 ← topics（关联款）

launch_waves（波段 ID）
   └── 关联 → tasks（所属波段）

okrs（OKR ID）
   └── 关联 → approvals（间接）
```

## 创建顺序

1. 先建 `products`（核心实体）
2. 再建 `launch_waves`、`tasks`（生产环节）
3. 再建 `platform_sales`、`stock_alerts`（销售环节）
4. 再建其他独立或弱关联的表
