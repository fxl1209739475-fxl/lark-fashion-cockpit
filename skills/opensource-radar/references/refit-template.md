# 改造方案模板（DeepSeek 输出参照）

每个开源项目的"改造思路"必须按这个模板填，确保可执行。

## 模板

```markdown
### [项目名] · 相关度 X/10 · ⭐ N · 工时 Yh

**应用场景**：[女装直接用 | 电商通用 | 管理工具 | AI 基础设施]

**核心能力**：[1 句话描述这项目能做啥]

**改造思路**（必须可执行，不要空话）：
1. **第 1 步**：具体动作（接什么 API / 改什么字段 / 替换什么逻辑）
2. **第 2 步**：...
3. **第 3 步**：...

**联动 cockpit skill**：列出 1-3 个会受影响的现有 skill

**风险提示**（如有）：
- 项目还在快速迭代（API 不稳）
- 需要 GPU / 付费 API
- 跟现有 X skill 有冲突

**工时拆解**：
- 调研 + 跑通 demo: Nh
- 接入 cockpit: Nh
- 测试 + 文档: Nh
- 合计: Yh
```

## 反模式（评估时禁止输出）

❌ "可以参考这个项目的设计思路" → 太虚，不可执行
❌ "对女装电商有借鉴意义" → 没说具体怎么用
❌ "可以集成到我们的系统" → 没说怎么集成
❌ "这是个好项目" → 没用

## 正向案例

✅ "把 OpenAgentX 替代 cockpit 现有的 deepseek 直调代码，把 32 个 sub-skill 注册成 tool，改 ask-boss.py / launch-decision.ps1 / product-matching.ps1 三个文件即可"

✅ "FashionAttr 接 01_产品库主图字段 → 推理输出标签 → 写 元素标签 字段。需要 PyTorch + GPU，建议租云 GPU 跑批量推理"

✅ "AutoLivePlanner 输入 02 表 GMV + 03 库存 + 主播档期 → 输出 24 表 计划 字段。改 4 个字段，加 1 个定时任务"
