# 会议精准传达者 — 信息不再损耗

> **🎯 用户洞察（5/4 傍晚）：** "传统会议依赖人传人，受时间/空间/表达能力/个人关系影响，<b>下层接到任务时一脸懵</b>，不知道前因后果。"

> **🚀 解决：** 会议 → AI 拆通用 + N 个角色定制 → 每人收到专属简报（含 why/how/when/notes 完整上下文）→ 同步建任务带说明。

> **板块：** 🅴 公司管理（第 26 个能力，task-lifecycle 升级版）  
> **数据源：** 飞书 vc + minutes API（真实妙记）  
> **下游：** task-collaboration / task-lifecycle 派任务

---

## 一、为什么需要这个 skill

| 痛点 | 旧方案 | 新方案 |
|---|---|---|
| 顶层开会下层不知道 | 中层口头传达 | AI 自动给每个角色生成专属简报 |
| 派任务下属一脸懵 | 任务标题 + 截止 | 任务 + **why（为什么）+ how（怎么做）+ when + notes** |
| 传达受表达能力影响 | 人传人逐层失真 | AI 直接传，质量 100% 一致 |
| 全员看同一份纪要 | 重要信息淹没 | 每人看到的下半部分**针对自己岗位** |

**核心价值：信息从领导到一线员工无损耗传递。**

---

## 二、架构

```
会议结束（飞书会议）
    ↓
拉妙记 AI 摘要 + 章节 + todos（vc + minutes API）
    ↓
AI 拆解：
    ├─ 通用部分（前 8 章节核心，所有人都看）
    └─ 角色定制部分（按 team-config 角色 keywords 匹配相关章节）
    ↓
为每个角色生成定制飞书文档：
    上半部分（相同）：通用 5-8 条核心要点
    下半部分（不同）：
        ├─ 跟你岗位相关的章节
        ├─ 你的具体待办（why / how / when / notes）
        └─ 任务上下文（为什么做、对完成人的价值）
    ↓
同步派任务（带任务说明引用文档段落）
    ↓
私信通知（卡片含文档链接 + 任务 ID）
```

## 三、角色 keywords 匹配规则

每个角色配 6-10 个关键词，用于自动匹配章节相关性：

| 角色 | Keywords |
|---|---|
| 设计师 | 设计 / 打版 / 版型 / 尺码 / 腰围 / 样品 / 款式 / 面料 |
| 生产主管 | 生产 / 工厂 / 大货 / 面料 / 采购 / 交期 / 产能 / 成品率 / 打样 |
| 内容编辑 | 内容 / 文案 / 拍摄 / 视频 / 直播 / 详情页 / 客服话术 / 种草 / 投流 |
| 客服 | 客服 / 投诉 / 退换货 / 满意度 / 咨询 |
| 主播 | 主播 / 直播间 / 选品 / 话术 / GMV |

匹配逻辑：章节 title 或 summary 含任一 keyword → 视为对该角色相关。

## 四、Why/How/When/Notes 模板

每个待办任务必须填这 4 个字段：

```
✅ 任务：DRS-2026-RM 设计稿出图

  Why：launch-decision AI 已基于历史爆款 DRS-0429-FL 建议下单 352 件，
       需要你先出 V1 设计稿启动 11 步流程
       
  How：参考 DRS-0429-FL 法式收腰版型 + 莓果红色卡 + 雪纺面料
       腰围必须放大 1cm（M 码 64→65.5cm），因为退货 3 次都是"按平时尺码买偏小"
       
  When：5/9 18:00 截止（已写入飞书日历）
  
  Notes：设计完发申丽媛打样，同步抄送朱健豪准备拍摄方案
```

## 八、和 task-lifecycle 的差别

| task-lifecycle（已有）| meeting-broadcaster（新）|
|---|---|
| 拉妙记 → 提取 todos → 派任务 | 拉妙记 → AI 拆通用+定制 → **每人专属简报** + 任务带上下文 |
| "你要做这件事" | "你要做这件事，**因为...，应该这样...，截止..., 注意...**" |
| 1 份纪要 + N 个任务 | **N 份个性化简报**（同会议每人看到不同）+ N 个任务 |
