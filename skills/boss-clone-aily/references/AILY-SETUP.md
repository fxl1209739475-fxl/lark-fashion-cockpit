# 飞书 Aily 智能伙伴 5 分钟搭建手册

> 把 cockpit 的「老板娘分身」从 Python 脚本升级成飞书原生 AI 助手。员工搜「老板娘 AI」就能聊。

## 第 1 步：进入 Aily 后台（30 秒）

1. 打开 https://aily.feishu.cn/
2. 用你的飞书账号登录（管理员身份）
3. 点 **「+ 创建应用」**

## 第 2 步：建一个 Aily 应用（1 分钟）

填表：
- **应用名称**：`老板娘 AI · lark-fashion-cockpit`
- **应用图标**：随便（建议用 👑 或上传一张老板娘头像）
- **应用描述**：`老板娘的 AI 数字分身。基于她的 200 题问答 + 5 条教训库，模拟她的下单决策风格`
- **可见范围**：先选「仅自己」测试，等通了再放给团队

点「创建」。

## 第 3 步：配置「角色」（人格）（1 分钟）

进入应用后点 **「角色」** → **「编辑」**，复制粘贴下面这段到「角色描述」：

```
你是 [老板娘真名] 的 AI 数字分身。她经营一家女装店，负责所有重大决策。

你的判断风格特征（来自她本人的 200 题问答）：
- 风险偏好：[等编译完 boss-persona.md 后填，先放占位]
- 选款直觉：偏经典款 + 显瘦剪裁
- 备货决策：宁少勿多，新品首单 50-200 件试水
- 客户/团队：直接、不绕弯，重视效率
- 红线：[填 G 维度的价值观]

回答原则：
1. 先给结论（一句话），再 1-3 句简短理由
2. 直接、有立场，不要打太极
3. 极端情况（破例 / 边界）要明确说红线
4. 信息不足时说出你需要什么再判断（不要瞎编）
5. 不要说"作为 AI"或"我是分身"，就以她本人语气回答

你能调用的工具：
- 查 26_老板语料库表（拉历史相关问答）
- 查 08_教训库（避坑提醒）
- 一键建任务 / 起审批
- 查 01_产品库 元素标签 / 02_4平台销售 历史数据
```

## 第 4 步：关联数据知识（最关键，2 分钟）

点 **「数据」** → **「+ 添加知识」**：

### 4a. 关联 26_老板语料库
- 类型：**飞书多维表格**
- Base：`LWsdbVtIaa2MaDsANm3cNdYgn1j`（lark-fashion-cockpit 主 base）
- 表：`26_老板语料库 (tblroqP4Kuz8a1yq)`
- 字段映射：
  - **问题**：维度 + 题目（拼起来）
  - **答案**：老板回答
  - **元数据**：题型 / 状态

### 4b. 关联 08_教训库（如果已建）
同上方式，把每条教训作为一条知识。

### 4c. 关联 14_审批记录
让分身能看老板历史审批决策。

## 第 5 步：（可选）配置技能（1 分钟）

点 **「技能」** → 添加技能：

### 技能 1: 下单决策复审
- 名称：`launch-decision-review`
- 触发：用户问"该备多少件"时
- 入参：`product_id` / `fans_estimate` / `target_audience`
- 输出：`{recommended_qty, size_ratio, color_ratio, reason, risks}`
- 实现：连接 `launch-decision.ps1`（Webhook 模式）

### 技能 2: 一键建任务
- 名称：`create-task-on-approval`
- 触发：用户说"那就照建议来"
- 实现：调 `task-collaboration` skill 的 API

## 第 6 步：发布 + 拿 app_id（30 秒）

点 **「发布」** → 选「全部成员」或「指定成员」。

发布后，应用顶部的 URL 里有 **app_id**（形如 `spring_xxxxx`）。复制下来发给 Claude（cockpit 后端代码要用）。

## 第 7 步：测试（30 秒）

在飞书里搜你刚才的应用名 `老板娘 AI · lark-fashion-cockpit` → 找到机器人 → 直接对话：

```
你: DRS-0429-FL 这款备 500 件还是 1000 件？
老板娘 AI: [基于 26_语料库 + 08_教训库 给出回答]
```

## 常见卡点

**Q：找不到「+ 创建应用」按钮？**
A：你的飞书租户可能还没开通 Aily 能力。去飞书管理员后台 https://www.feishu.cn/admin/aily 申请开通。

**Q：数据知识关联不上多维表格？**
A：检查 lark-fashion-cockpit 主应用的 base 权限是否给了 Aily 应用。在主应用「权限管理」加上"读取多维表格"。

**Q：角色描述写得不知道怎么填？**
A：先用 cockpit 现有的 `personas/boss-persona.md` 直接粘贴（老板娘答完语料后跑 `python scripts/build-boss-persona.py` 自动生成）。
