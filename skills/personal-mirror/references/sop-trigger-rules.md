# SOP 沉淀触发规则 — 什么时候该写 SOP

`personal-mirror` 检测员工日常飞书行为，达到以下条件之一就触发"该沉淀 SOP"建议。

## 触发条件清单（任一满足）

### 条件 1：同关键词 ≥ 3 次（最常用）
- 同一个产品 ID 在 7 天内被员工提及 ≥ 3 次
- 同一个客户名 / 工厂代号 / 问题术语 ≥ 3 次
- 同样的 sku 改稿请求被设计师收到 ≥ 3 次

### 条件 2：同事反复来同类需求 ≥ 3 次
- 工厂主管 7 天内 5 次找设计师对色卡 → 触发"色卡 SOP"
- 客服 7 天内 4 次问运营"DRS-XXX 库存" → 触发"库存查询 SOP"
- 主播 7 天内 3 次问"这场直播带啥款" → 触发"带款选品 SOP"

### 条件 3：历史 SOP 库已有但今天没用上
- 员工 SOP 库里有"色卡确认 3 步法"
- 今日发现员工又在跟工厂手动对色卡（没用 SOP）
- → 提示"你 SOP 库有这个，怎么没用？"

### 条件 4：工时反复异常
- 员工"DRS-XXX 改稿"任务 3 次都超过预估 50%
- → 触发"改稿超时根因分析 SOP"

### 条件 5：同一个错误反复犯
- 7 天内工厂 3 次反馈"色差"
- 7 天内客服 3 次解释"为啥不发某色"
- → 触发"色彩管理 SOP"

---

## 不触发的情况（避免噪音）

❌ 出现 1-2 次（偶发，不需要沉淀）
❌ 关键词太通用（"今天"、"OK" 这类）
❌ 已沉淀过（去重检测）

---

## 阈值建议

```python
SOP_TRIGGER_THRESHOLDS = {
    "keyword_repeat_in_7d": 3,        # 关键词重复
    "same_topic_request_7d": 3,        # 同主题请求次数
    "task_overrun_count": 3,           # 任务超时次数
    "error_repeat_7d": 3,              # 同类错误重复
    "min_keyword_length": 3,           # 关键词最少 3 字（过滤 OK / 嗯 等）
}
```

```
{触发主题} SOP

## 适用场景
（AI 从历史对话提炼）

## 标准操作步骤
1.
2.
3.

## 常见问题
- Q1:
- A1:

## 触发记录（自动）
- {date1}: {situation1}
- {date2}: {situation2}
- {date3}: {situation3}

## 预计省时
每次 X 分钟 × 月均 Y 次 = Z 分钟/月
```

## 沉淀去向（员工个人，不是公司）

| 沉淀位置 | 用途 |
|---|---|
| 员工个人飞书文档「我的 SOP 库 - YYYYMM」| 长文档形式存详细 SOP |
| 员工个人飞书 wiki「我的成长档案」| 体系化分类 |
| 员工个人 Lingo 词典 | 术语短解释 |

⚠️ **不写公司公共 wiki / base** — 这是员工私域，员工自己决定要不要分享给团队。
