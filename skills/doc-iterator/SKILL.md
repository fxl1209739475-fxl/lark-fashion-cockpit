# 评论驱动文档迭代 — 一句话出 V2

> **🎯 用户洞察：** "团队评论文档后，自己手动整理改进很累。能不能让 AI 自动整合评论 → 输出 V2 → 在评论区回'已采纳'？"

> **🚀 解决：** 拉文档 + 拉评论 → AI 整合（修改/注意/保持/改进 4 类）→ 追加 V2 修订段落 → 评论回复"已采纳" → 群推 V2 review 卡片。

> **板块：** 🅴 公司管理（第 27 个能力，跨企业普适）  
> **数据源：** lark-cli docs +fetch + drive file.comments list  
> **输出：** docs +update append + drive +add-comment 回复

| 法律合同各方意见 | 法务人工合并 | AI 自动合并 |

**核心价值：文档协作不再是"人工合并瓶颈"。**

---

## 二、5 步工作流

```
触发：用户说"根据评论改一下设计稿" / "迭代一下"
        ↓
Step 1: lark-cli docs +fetch          拉文档当前内容
        ↓
Step 2: lark-cli drive file.comments list  拉所有未解决评论
        ↓
Step 3: AI 整合（规则化 + DeepSeek 兜底）
        - 把评论分类：修改 / 注意 / 保持 / 改进
        - 提取关键修订项（V1 → V2 对比表）
        ↓
Step 4: lark-cli docs +update --command append  追加 V2 修订段落
        - V2 修订记录（按类型分组）
        - 关键修订对比表（V1 → V2）
        - 下一步行动建议
        ↓
Step 5: lark-cli drive +add-comment  评论区回复"✅ 已采纳"
        + 群里推 V2 review 卡片
```

---

## 三、AI 整合规则（4 类分流）

| 类型 | 触发关键词 | V2 处理 |
|---|---|---|
| 🔧 修改 | "建议放大/改/必须/要" | V2 应用修改 |
| ⚠️ 注意 | "注意/关注/提醒/防止" | V2 加注意事项 |
| ✅ 保持 | "OK/沿用/预估/不错" | V2 保留原方案 |
| 📌 改进 | 其他建设性意见 | V2 加改进点 |

## 四、实测演示（2026-05-04）

### 输入

文档：DRS-2026-RM 设计稿（5 条 AI 评论）：
1. 腰围 M=64cm 偏小 → 建议放大 65.5cm
2. 莓果红色卡 → 寄回老板确认 + 工厂签字
3. 蕾丝面料 → 详情页加洗护说明
4. 尺码占比 → 关注 XL 历史均偏高 12%
5. 整体设计 OK → 沿用法式收腰 + 雪纺

### 输出

V2 修订段落自动追加，含：
- 5 条评论按类型分类（修改 2 / 改进 1 / 注意 1 / 保持 1）
- V1 → V2 对比表（5 行）
- 下一步行动建议
- 5 条"✅ 已采纳"自动回复

每条评论自动回"✅已采纳，已合并到 V2 修订段落"。

## 七、运行方式

```bash
# 单文档迭代
PowerShell -File scripts/doc-iterator.ps1 -DocId <doc_token>

# 干跑（不真改文档）
PowerShell -File scripts/doc-iterator.ps1 -DryRun

# 跟 cron 联动（每天 18:00 扫所有有未解决评论的文档自动迭代）
0 18 * * * powershell -File scripts/doc-iterator.ps1
```
